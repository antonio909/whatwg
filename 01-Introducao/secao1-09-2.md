<h1>1.9.2 Convenções tipográficas</h1>

<p>Isto é uma definição, requisito ou explicação.</p>

<dl>
  <dt>Nota</dt>
  <dd>Esta é uma nota.</dd>
</dl>

<dl>
  <dt>Exemplo</dt>
  <dd>Este é um exemplo.</dd>
</dl>

<dl>
  <dt>Issue</dt>
  <dd>Isto é uma questão em aberto.</dd>
</dl>

<dl>
  <dt>Aviso!</dt>
  <dd>Isto é um aviso.</dd>
</dl>

<dl>
  <dt>IDL</dt>
  <dd>
    <code>
      [Exposed=Window]
      interface Example {
        // Esta é uma definição IDL
      };
    </code>
  </dd>
</dl>

<dl>
  <dt>Para desenvolvedores web (não normativo)</dt>
  <dd>
    <code>variable = object.method([optionalArgument])</code>
    Esta é uma nota aos autores descrevendo o uso de uma interface.
  </dd>
</dl>

<dl>
  <dt>CSS</dt>
  <dd>
    <code>/* Este é um fragmento de CSS */</code>
  </dd>
</dl>

<p align="justify">
  A instância de definição de um termo é marcada desta forma.
  As utilizações desse termo são marcadas desta ou daquela forma.
</p>

<p align="justify">
  A instância de definição de um elemento, atributo ou API é marcada desta forma.
  As referências a esse elemento, atributo ou API são marcadas desta forma.
</p>

<p>
  Outros fragmentos de código são marcados desta forma.
</p>

<p>As variáveis são marcadas desta forma.</p>

<p>Em um algoritmo, as etapas em seções síncronas são marcadas com ⌛.</p>
